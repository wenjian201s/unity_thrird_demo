# 基于 Unity URP 的分区域角色 NPR 渲染方案技术文档

## 摘要

本文整理的是一套运行于 **Unity URP** 的自研角色风格化渲染方案。该方案的目标不是追求物理正确的写实渲染，而是构建一套 **风格统一、层次稳定、脸部阴影可控、轮廓清晰** 的角色 NPR（Non-Photorealistic Rendering）系统。

从工程结构上看，这套方案由四部分组成：

1. Shader 主文件：负责材质参数面板、关键字、Pass 组织与渲染状态管理。
2. 输入定义文件：负责贴图、采样器、材质常量缓冲的统一声明。
3. 主体着色 Pass：负责主体颜色、间接光、主光、Ramp、高光、丝袜、边缘光、自发光、脸部修正等风格项合成。
4. 描边 Pass：负责模型外扩与描边颜色输出。

整套方案的核心特征可以概括为：

- 角色按 **脸 / 头发 / 上半身 / 下半身** 分区域处理
- 主体阴影通过 **LightMap + 阈值控制 + Ramp 映射** 完成卡通化
- 脸部使用 **头部坐标系 + FaceMap + SDF 阴影** 实现稳定脸影
- 描边采用 **背面外扩法**，并支持 **平滑法线描边**
- 通过 **丝袜、边缘光、自发光、假描边** 等模块增强角色风格表现

---

## 1. 渲染目标与设计原则

### 1.1 渲染目标

该方案希望实现的角色观感包括：

- 明暗层次具有明显的二次元分层感
- 头发、皮肤、衣物、配件在同一 Shader 框架下保持不同材质特征
- 脸部阴影稳定，不因普通法线受光而出现突兀变化
- 描边宽度在不同镜头距离下保持较稳定的屏幕表现
- 支持常见角色渲染所需的阴影、深度、深度法线、描边、叠加重绘等管线功能

### 1.2 设计原则

本方案不是 PBR 的调参版本，而是：

- 使用现代引擎的坐标、光照、深度、阴影接口
- 使用贴图编码和关键字分区组织材质信息
- 使用风格化的颜色映射和模块化叠加来替代统一 BRDF

因此，这是一套 **以美术控制优先于物理正确** 的角色 NPR 系统。

## 2. 整体架构

## 2.1 Shader 主文件的职责

Shader 主文件负责声明材质面板、注册关键字、组织 Pass，并把真正的逻辑分发到不同的 HLSL 文件中。

### 关键字与区域系统

```hlsl
[KeywordEnum (None, Face, Hair, UpperBody, LowerBody)] _Area("Material area", float) = 0

#pragma shader_feature_local _AREA_FACE
#pragma shader_feature_local _AREA_HAIR
#pragma shader_feature_local _AREA_UPPERBODY
#pragma shader_feature_local _AREA_LOWERBODY
```

这意味着同一份 Shader 代码可以在编译时生成不同区域变体，并在运行时由材质实例切换。

### Pass 结构

```hlsl
Pass
{
    Name "ShadowCaster"
    Tags { "LightMode" = "ShadowCaster" }
}

Pass
{
    Name "DepthOnly"
    Tags { "LightMode" = "DepthOnly" }
}

Pass
{
    Name "DepthNormals"
    Tags { "LightMode" = "DepthNormals" }
}

Pass
{
    Name "DrawCore"
    Tags
    {
        "RenderPipeline" = "UniversalPipeline"
        "RenderType" = "Opaque"
    }
}

Pass
{
    Name "DrawOverlay"
    Tags
    {
        "RenderPipeline" = "UniversalPipeline"
        "RenderType" = "Opaque"
        "LightMode" = "UniversalForward"
    }
}

Pass
{
    Name "DrawOutline"
    Tags
    {
        "RenderPipeline" = "UniversalPipeline"
        "RenderType" = "Opaque"
        "LightMode" = "UniversalForwardOnly"
    }
}
```

### 架构含义

- `ShadowCaster`：角色参与主光阴影投射
- `DepthOnly`：角色可写入深度
- `DepthNormals`：角色可输出深度法线供 SSAO / 后处理使用
- `DrawCore`：主体风格化角色着色
- `DrawOverlay`：额外叠加绘制层
- `DrawOutline`：独立描边绘制层

这一步确立的是完整角色材质的 **工程骨架**。

## 4. 顶点阶段：准备风格化着色所需的数据

主体着色的顶点阶段实现如下：

```hlsl
Varyings StarRailPassVertex(Attributes input)
{
    Varyings output = (Varyings)0;

    VertexPositionInputs positionInputs = GetVertexPositionInputs(input.positionOS.xyz);
    VertexNormalInputs normalInputs = GetVertexNormalInputs(input.normalOS, input.tangentOS);
    output.uv = TRANSFORM_TEX(input.texcoord, _BaseMap);
    output.positionWSAndFogFactor = float4(positionInputs.positionWS, ComputeFogFactor(positionInputs.positionCS.z));
    output.normalWS = normalInputs.normalWS;
    output.viewDirectionWS = unity_OrthoParams.w == 0 ? GetCameraPositionWS() - positionInputs.positionWS : GetWorldToViewMatrix()[2].xyz;
    output.SH = SampleSH(lerp(normalInputs.normalWS, float3(0,0,0), _IndirectLightFlattenNormal));
    output.positionCS = positionInputs.positionCS;
    
    return output;
}
```

这一阶段准备了：

- UV
- 世界空间位置
- 世界空间法线
- 世界空间视线方向
- 球谐环境光 SH
- 雾因子
- 裁剪空间位置

其中比较重要的是：

```hlsl
output.SH = SampleSH(lerp(normalInputs.normalWS, float3(0,0,0), _IndirectLightFlattenNormal));
```

这说明间接光并不是严格沿真实法线计算，而是允许通过 `_IndirectLightFlattenNormal` 压平法线，从而减少环境光中的细碎噪声，使角色整体受光更平稳。

## 5. 间接光阶段：用 SH + AO 先建立角色体积基础

主体 Pass 中，间接光的实现如下：

```hlsl
float3 indirectLightColor = input.SH * _IndirectLightUsage;
#if _AREA_HAIR || _AREA_UPPERBODY || _AREA_LOWERBODY
    indirectLightColor *= lerp(1, lightMap.r, _IndirectLightOcclusionUsage);
#else
    float faceMask = lerp(faceMap.g, 1, step(faceMap.r, 0.5));
    indirectLightColor *= lerp(1, faceMask, _IndirectLightOcclusionUsage);
#endif
indirectLightColor *= lerp(1, baseColor.rgb, _IndirectLightMixBaseColor);
```

### 技术目的

- 用球谐环境光给角色暗部打底
- 用 AO 控制暗部区域
- 用底色混合避免环境光发灰

### 区域差异

- 身体 / 头发：AO 来自 `lightMap.r`
- 脸部：AO 来自 `faceMap.g` 与 `faceMap.r` 的组合遮罩

间接光阶段不追求复杂，而是确保角色在没有主光直接照亮时仍然具有稳定、可控的基础体积感。

---

## 6. 主体受光阶段：用阈值化 Lambert 生成 Toon 阴影驱动值

对于头发和身体，主光阴影并不直接采用 `NoL * BaseColor`，而是先生成一个风格化阴影驱动值。

```hlsl
float NoL = dot(normalWS, lightDirectionWS);
float remappedNoL = NoL * 0.5 + 0.5;
mainLightShadow = smoothstep(1 - lightMap.g + _ShadowThresholdCenter - _ShadowThresholdSoftness,
                             1 - lightMap.g + _ShadowThresholdCenter + _ShadowThresholdSoftness,
                             remappedNoL);
mainLightShadow *= lightMap.r;
```

### 技术含义

- `NoL` 只是一个基础输入
- `lightMap.g` 决定局部阴影边界位置
- `_ShadowThresholdCenter` 控制整体阴影线前后移动
- `_ShadowThresholdSoftness` 控制过渡边缘软硬
- `lightMap.r` 再次压暗某些区域

这说明你的主体阴影不是简单漫反射，而是 **“受光输入 → 阈值控制 → 风格化阴影值”** 的流程。

---

## 7. Ramp 映射阶段：主光颜色风格化的核心

在得到 `mainLightShadow` 后，Shader 会将其映射到 Ramp 贴图。

```hlsl
float rampUVx = mainLightShadow * (1 - _ShadowRampOffset) + _ShadowRampOffset;
float rampUVy = (2 * rampRowIndex + 1) * (1.0 / (rampRowNum * 2));
float2 rampUV = float2(rampUVx, rampUVy);
```

### 头发与身体使用不同 Ramp

```hlsl
#if _AREA_HAIR
    coolRamp = SAMPLE_TEXTURE2D(_HairCoolRamp, sampler_HairCoolRamp, rampUV).rgb;
    warmRamp = SAMPLE_TEXTURE2D(_HairWarmRamp, sampler_HairWarmRamp, rampUV).rgb;
#elif _AREA_FACE || _AREA_UPPERBODY || _AREA_LOWERBODY
    coolRamp = SAMPLE_TEXTURE2D(_BodyCoolRamp, sampler_BodyCoolRamp, rampUV).rgb;
    warmRamp = SAMPLE_TEXTURE2D(_BodyWarmRamp, sampler_BodyWarmRamp, rampUV).rgb;
#endif
```

### 冷暖混合

```hlsl
float isDay = lightDirectionWS.y * 0.5 + 0.5;
float3 rampColor = lerp(coolRamp, warmRamp, isDay);
mianLightColor *= baseColor.rgb * rampColor;
```

### 技术意义

- 阴影色不是数学算出来的，而是由美术绘制的 Ramp 贴图决定
- 头发、皮肤、衣物可以拥有不同色带风格
- 冷暖 ramp 混合使整体色调更灵活

这一步实际上把主光从“亮度问题”变成了“风格色带采样问题”。

---

## 8. 材质分层阶段：LightMap.a 驱动多行 Ramp

身体区域还使用了 `lightMap.a` 作为材质编号来选择 Ramp 行。

```hlsl
int rawIndex = (round((lightMap.a + 0.0425) / 0.0625) - 1) / 2;
rampRowIndex = lerp(rawIndex, rawIndex + 4 < 8 ? rawIndex + 4 : rawIndex + 4 - 8, fmod(rawIndex, 2));
```

### 技术意义

- 一张 BodyRamp 中可以存放多行不同材质色带
- 通过 `lightMap.a`，同一 Shader 可以在同一个身体区域中区分不同材质
- 比如布、皮革、金属配件、皮肤边缘都能走不同的 toon 色带

这一步本质上是把 **材质分类信息** 前置编码到了贴图中。

## 9. 脸部专用阶段：头部坐标系 + FaceMap + SDF 阴影

脸部不走普通 Lambert，而使用专门的 SDF 阴影方案。

```hlsl
float3 headForward = normalize(_HeadForward);
float3 headRight = normalize(_HeadRight);
float3 headUp = normalize(cross(headForward, headRight));

float3 fixedLightDirectionWS = normalize(lightDirectionWS - dot(lightDirectionWS, headUp) * headUp);
float2 sdfUV = float2(sign(dot(fixedLightDirectionWS, headRight)), 1) * input.uv * float2(-1, 1);
float sdfValue = SAMPLE_TEXTURE2D(_FaceMap, sampler_FaceMap, sdfUV).a;
sdfValue += _FaceShadowOffset;

float sdfThreshold = 1 - (dot(fixedLightDirectionWS, headForward) * 0.5 + 0.5);
float sdf = smoothstep(sdfThreshold- _FaceShadowTransitionSoftness, sdfThreshold + _FaceShadowTransitionSoftness, sdfValue);
mainLightShadow = lerp(faceMap.g, sdf, step(faceMap.r, 0.5));
```

### 技术意义

- 使用 `_HeadForward`、`_HeadRight` 建立头部局部坐标系
- 将光方向投影到头部水平面，避免上下翻转时脸影错误
- 使用 `FaceMap.a` 作为 SDF 阴影值
- 使用 `FaceMap.g` 处理五官或口腔 AO 替换区域

这一阶段的目标是让脸部阴影摆脱普通法线受光带来的不稳定，而转向一套 **可控、稳定、符合角色审美的脸部阴影系统**。

## 10. 高光阶段：材质分类高光，而不是统一 PBR 高光

高光部分的实现如下：

```hlsl
float3 halfVectorWS = normalize(viewDirectionWS + lightDirectionWS);
float NoH = dot(normalWS, halfVectorWS);
float blinnPhong = pow(saturate(NoH), _SpecularExpon);

float nonMetalSpecular = step(1.04 - blinnPhong, lightMap.b) * _SpecularKsNonMetal;
float metalSpecular = blinnPhong * lightMap.b * _SpecularKsMetal;
```

### 金属度判定

```hlsl
float metallic = 0;
#if _AREA_UPPERBODY || _AREA_LOWERBODY
    metallic = saturate((abs(lightMap.a - _SpecularMetalRange) - 0.1) / (0 - 0.1));
#endif

specularColor = lerp(nonMetalSpecular, metalSpecular * baseColor.rgb, metallic);
specularColor *= mainLight.color;
specularColor *= _SpecularBrightness;
```

### 技术意义

- 非金属高光偏离散，更接近卡通反光块
- 金属高光更连续，并且带材质本色
- `lightMap.a` 同时参与了材质类型判定

这是一种典型的 **风格化材质高光** 实现方式。

## 11. 丝袜 / 半透材质阶段：视角驱动的渐变材质层

丝袜效果由 `NoV` 驱动，并结合渐变函数实现。

```hlsl
float NoV = dot(normalWS, viewDirectionWS);
float fac = NoV;
fac = pow(saturate(fac), _StockingsTransitionPower);
fac = saturate((fac - _StockingsTransitionHardness / 2) / (1- _StockingsTransitionHardness));
fac = fac * (stockingsMapB * _StockingsTextureUsage + (1 - _StockingsTextureUsage));
fac = lerp(fac, 1, stockingsMapRG.g);
```

### 三段渐变色带

```hlsl
Gradient curve = GradientConstruct();
curve.colorsLength = 3;
curve.colors[0] = float4(_StockingsDarkColor.rgb, 0);
curve.colors[1] = float4(_StockingsTransitionColor.rgb, _StockingsTransitionThreshold);
curve.colors[2] = float4(_StockingsLightColor.rgb, 1);
float3 stockingsColor = SampleGradient(curve, fac);

stockingsEffect = lerp(1, stockingsColor, stockingsMapRG.r);
```

### 技术意义

- 使用 NoV 决定视角相关的亮暗变化
- 使用高频纹理添加局部细节
- 使用 3 点 Gradient 实现暗 → 过渡 → 亮 的风格材质色带

这是一个独立的 **局部材质风格模块**，而不是统一光照模型的一部分。

## 12. 边缘光阶段：深度驱动的屏幕空间轮廓光

边缘光实现如下：

```hlsl
float linearEyeDepth = LinearEyeDepth(input.positionCS.z, _ZBufferParams);
float3 normalVS = mul((float3x3)UNITY_MATRIX_V, normalWS);
float2 uvOffset = float2(sign(normalVS.x), 0) * _RimLightWidth / (1 + linearEyeDepth) / 100;
int2 loadTexPos = input.positionCS.xy + uvOffset * _ScaledScreenParams.xy;
loadTexPos = min(loadTexPos, _ScaledScreenParams.xy - 1);
float offsetSceneDepth = LoadSceneDepth(loadTexPos);
float offsetLinearEyeDepth = LinearEyeDepth(offsetSceneDepth, _ZBufferParams);
float rimLight = saturate(offsetLinearEyeDepth - (linearEyeDepth + _RimLightThreshold)) / _RimLightFadeout;
```

### 着色

```hlsl
float3 rimLightColor = rimLight * mainLight.color.rgb;
rimLightColor *= _RimLightTintColor;
rimLightColor *= _RimLightBrightness;
```

### 技术意义

- 不是普通 Fresnel，而是通过场景深度差检测轮廓边缘
- 只沿视图空间法线左右方向偏移采样
- 描边宽度随深度自动缩放，形成近粗远细

这种 rim 更接近“屏幕空间轮廓发光”，而不是全表面泛边。

## 13. 发光与脸部假描边：风格增强层

## 13.1 自发光

```hlsl
float3 emissionColor = 0;
#if _EMISSION_ON
    emissionColor = areaMap.a;
    emissionColor *= lerp(1, baseColor, _EmissionMixBaseColor);
    emissionColor *= _EmissionTintColor;
    emissionColor *=_EmissionIntensity;
#endif
```

这里把 `areaMap.a` 当作发光 mask，再乘以底色、Tint 与强度。

## 13.2 脸部假描边

```hlsl
float fakeOutlineEffect = 0;
float3 fakeOutlineColor = 0;
#if _AREA_FACE && _OUTLINE_ON
    float fakeOutline = faceMap.b;
    float3 headForwardShadow = normalize(_HeadForward);
    fakeOutlineEffect = smoothstep(0.0, 0.25, pow(saturate(dot(headForwardShadow, viewDirectionWS)), 20) * fakeOutline);
    float2 outlineUV = float2(0, 0.0625);
    float3 coolRampShadow = SAMPLE_TEXTURE2D(_BodyCoolRamp, sampler_BodyCoolRamp, outlineUV).rgb;
    float3 warmRampShadow = SAMPLE_TEXTURE2D(_BodyWarmRamp, sampler_BodyWarmRamp, outlineUV).rgb;
    float3 ramp = lerp(coolRampShadow, warmRampShadow, 0.5);
    fakeOutlineColor = pow(saturate(ramp), _OutlineGamma);
#endif
```

### 技术意义

- 自发光为角色提供局部风格亮点
- 假描边用于补足脸部在某些角度下的线稿感
- 两者都属于“风格增强层”，不是基础光照骨架

## 14. 最终颜色合成阶段

主体最终颜色采用层叠合成方式：

```hlsl
float3 albedo = 0;
albedo += indirectLightColor;
albedo += mianLightColor;
albedo += specularColor;
albedo *= stockingsEffect;
albedo += rimLightColor * lerp(1, albedo, _RimLightMixAlbedo);
albedo += emissionColor;
albedo = lerp(albedo, fakeOutlineColor, fakeOutlineEffect);
```

最终再处理 Alpha、Clip 与雾：

```hlsl
float4 color = float4(albedo, alpha);
clip(color.a - _AlphaClip);
color.rgb = MixFog(color.rgb, input.positionWSAndFogFactor.w);
return color;
```

### 技术意义

该方案最终并不是统一公式直出，而是一个 **分层式风格合成器**：

- 间接光打底
- 主光给主体层次
- 高光给材质感
- 丝袜做局部调制
- Rim 加轮廓亮边
- Emission 补亮点
- 假描边修脸部轮廓

## 15. 描边 Pass：独立轮廓系统

描边在主文件中是独立 Pass：

```hlsl
Pass
{
    Name "DrawOutline"
    ...
    Cull Front
    ZWrite [_ZWrite]
}
```

### 技术含义

- 使用 `Cull Front` 只渲染壳体背面
- 通过顶点外扩把模型膨胀成轮廓壳
- 与主体 Pass 分离，便于独立控制

---

## 16. 描边顶点阶段：法线 / 平滑法线外扩

```hlsl
float width = _OutlineWidth;
width *= GetOutlineCameraFovAndDistanceFixMultiplier(positionInputs.positionVS.z);

float3 positionWS = positionInputs.positionWS;
#if _OUTLINE_VERTEX_COLOR_SMOOTH_NORMAL
    float3x3 tbn = float3x3(normalInputs.tangentWS, normalInputs.bitangentWS, normalInputs.normalWS);
    positionWS += mul(input.color.rgb * 2 - 1, tbn) * width;
#else
    positionWS += normalInputs.normalWS * width;
#endif
```

### 技术意义

- 普通模式：沿法线外扩
- 高级模式：使用顶点色中编码的平滑法线方向外扩
- 支持相机距离 / FOV 修正，保持描边屏幕宽度稳定

### 相机修正函数

```hlsl
float GetOutlineCameraFovAndDistanceFixMultiplier(float positionVS_Z)
{
    float cameraMulFix;
    if(unity_OrthoParams.w == 0)
    {
        cameraMulFix = abs(positionVS_Z);
        cameraMulFix = ApplyOutlineDistanceFadeOut(cameraMulFix);
        cameraMulFix *= GetCameraFOV();
    }
    else
    {
        float orthoSize = abs(unity_OrthoParams.y);
        orthoSize = ApplyOutlineDistanceFadeOut(orthoSize);
        cameraMulFix = orthoSize * 50;
    }

    return cameraMulFix * 0.00005;
}
```

这一步解决的是：描边在不同相机距离和镜头焦距下不稳定的问题。

## 17. 描边颜色阶段：从 Ramp 暗部取色，而不是纯黑描边

描边片元如下：

```hlsl
#if _AREA_HAIR
    float2 outlineUV = float2(0, 0.5);
    coolRamp = SAMPLE_TEXTURE2D(_HairCoolRamp, sampler_HairCoolRamp, outlineUV).rgb;
    warmRamp = SAMPLE_TEXTURE2D(_HairWarmRamp, sampler_HairWarmRamp, outlineUV).rgb;
#elif _AREA_UPPERBODY || _AREA_LOWERBODY
    float4 lightMap = 0;
    #if _AREA_UPPERBODY
        lightMap = SAMPLE_TEXTURE2D(_UpperBodyLightMap, sampler_UpperBodyLightMap, input.uv);
    #elif _AREA_LOWERBODY
        lightMap = SAMPLE_TEXTURE2D(_LowerBodyLightMap, sampler_LowerBodyLightMap, input.uv);
    #endif
    float materialEnum = lightMap.a;
    float materialEnumOffset = materialEnum + 0.0425;
    float outlineUVy = lerp(materialEnumOffset, materialEnumOffset + 0.5 > 1 ? materialEnumOffset + 0.5 - 1 : materialEnumOffset + 0.5, fmod((round(materialEnumOffset/0.0625) - 1)/2, 2));
    float2 outlineUV = float2(0, outlineUVy);
    coolRamp = SAMPLE_TEXTURE2D(_BodyCoolRamp, sampler_BodyCoolRamp, outlineUV).rgb;
    warmRamp = SAMPLE_TEXTURE2D(_BodyWarmRamp, sampler_BodyWarmRamp, outlineUV).rgb;
#elif _AREA_FACE
    float2 outlineUV = float2(0, 0.0625);
    coolRamp = SAMPLE_TEXTURE2D(_BodyCoolRamp, sampler_BodyCoolRamp, outlineUV).rgb;
    warmRamp = SAMPLE_TEXTURE2D(_BodyWarmRamp, sampler_BodyWarmRamp, outlineUV).rgb;
#endif
```

最终颜色：

```hlsl
float3 ramp = lerp(coolRamp, warmRamp, 0.5);
float3 albedo = pow(saturate(ramp), _OutlineGamma);
```

### 技术意义

- 描边颜色不是固定黑线
- 头发、身体、脸部可以取不同的暗部色带
- 描边和主体阴影系统保持同一色彩体系

这使描边不会显得脱离主体材质，而是自然融入整体角色风格中。

## 19. 资源约定总结

根据当前代码，这套方案对应的资源约定如下：

### 19.1 BaseColor

- FaceColorMap
- HairColorMap
- UpperBodyColorMap
- LowerBodyColorMap

### 19.2 LightMap

建议按通道编码：

- R：AO / 阴影压制
- G：阴影阈值控制
- B：高光 Mask
- A：材质编号 / Ramp 行 / 金属判定参考

### 19.3 FaceMap

建议按通道编码：

- A：脸部 SDF 阴影值
- G：脸部 AO / 嘴部暗部
- B：假描边 Mask
- R：特殊区域遮罩辅助

### 19.4 Ramp

- HairCoolRamp / HairWarmRamp
- BodyCoolRamp / BodyWarmRamp

其中 BodyRamp 应支持多行图集。

### 19.5 Stockings

建议按通道编码：

- R：启用 Mask
- G：厚度 / 亮区控制
- B：高频细节扰动

## 20. 方案总结

这套自研角色渲染方案的技术本质可以总结为：

- 用 **URP 的标准渲染骨架** 承接角色材质
- 用 **分区域贴图系统** 区分脸、头发、衣物等不同材质逻辑
- 用 **LightMap 编码** 驱动阴影分界、材质分类和高光表现
- 用 **Ramp 映射** 统一主体色彩风格
- 用 **FaceMap + SDF** 实现稳定脸部阴影
- 用 **背面外扩描边** 实现角色轮廓
- 用 **丝袜、边缘光、自发光、假描边** 做风格增强

它不是写实角色渲染，也不是现成商业项目的照搬，而是一套明确服务于 **风格控制、角色表现、可扩展工程组织** 的自研 NPR 角色方案。

如果继续扩展，这套方案还可以进一步加入：

- 头发条带高光
- 眼睛专用材质层
- 角色局部透明分层
- 法线贴图与风格法线混合
- 面部表情驱动的光照修正
- 屏幕空间轮廓加强或后处理联动

从目前的实现来看，它已经具备一套完整、清晰且可持续扩展的角色风格渲染框架。
